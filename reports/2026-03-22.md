  - Authors: n/a
  - Affiliations: not provided
  - Describes **matmul epilogue fusion**: attaching post-GEMM elementwise work (e.g., bias add, activation, scaling, type conversion) to the matmul kernel so results are produced in-register/shared memory before the final writeback.
  - GPU benefit: **avoids an extra kernel launch and a full read/write of the output tensor**, reducing global memory bandwidth/latency and improving end-to-end GEMM throughput.
  - Practical guidance centers on selecting/configuring supported epilogues so common NN patterns (GEMM + bias + activation) run as **one fused GPU kernel**.
