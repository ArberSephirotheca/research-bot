- [Help with Transpose SharedMemoryKernel (r/CUDA)](https://www.reddit.com/r/CUDA/comments/1sidzlf/help_with_transpose_sharedmemorykernel/) — 2026-04-11
  - Authors: n/a
  - Affiliations: not provided
  - Shared-memory transpose kernels hinge on avoiding bank conflicts: use a padded tile (e.g., `TILE_DIM x (TILE_DIM+1)`) so the shared-memory write/read during transpose doesn’t serialize, while keeping global loads/stores coalesced.
  - Performance is typically limited by memory throughput, not compute; choose tile size and per-thread work (e.g., multiple rows per thread) to balance ILP and occupancy, and watch register/shared-memory usage so occupancy doesn’t collapse.
  - Relevance to GEMM: transpose is often used to improve coalescing for one operand (e.g., turning column-major access into row-major); the same tiling/padding principles apply to shared-memory staging in matrix multiplication to avoid bank conflicts and maximize bandwidth.
